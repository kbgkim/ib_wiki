# 통합 리스크 데이터 모델 (IB Risk Data Model)

## 🔥 목적

본 문서는 통합 IB 리스크 시스템의 핵심 데이터 아키텍처와 엔티티 간의 관계를 정의합니다. 
모든 자산군(Equity, PF, ABS, NPL)을 **'현금흐름(Cashflow)'**이라는 공통 분모로 통합하여 관리하는 프로덕션급 설계를 지향합니다.

### ─────────────

## 📌 핵심 운영 개념

### 1. 포지션 중심 (Position-Centric)
- 모든 리스크 산출과 데이터 집계는 **'포지션(Position)'** 단위에서 수행됨을 원칙으로 합니다.
- 하나의 딜(Deal)은 다수의 포지션을 가질 수 있으며, 리스크 변수(PD, LGD, EAD)는 포지션별로 부여됩니다.

### 2. 현금흐름 기반 (Cashflow-Based)
- 리스크의 본질을 **'미래 현금흐름의 불확실성'**으로 정의합니다.
- 부도(Default)는 예정된 현금흐름의 미실현으로 간주하며, 손실(Loss)은 실현된 현금흐름과 예정된 흐름의 차이(Shortfall)로 측정합니다.

### ─────────────

## 🧠 리스크 연결 구조 (Risk-Data Chain)

리스크 데이터는 원천 데이터에서 최종 지표까지 다음의 연쇄 반응을 거쳐 생성됩니다.

### 리스크 데이터 파이프라인
1. **Master Layer**: Deal, Counterparty, Collateral 데이터 로드  
2. **Position Layer**: 개별 투자 단위(Position)와 익스포저(Exposure) 확정  
3. **Execution Layer**: 시나리오별 예상 현금흐름(Cashflow) 생성  
4. **Logic Layer**: 현금흐름 부족(Shortfall) 및 회수(Recovery) 시뮬레이션  
5. **Result Layer**: 최종 PD, LGD, EAD 및 기대손실(EL) 산출  

### ─────────────

## ⚖️ 자산별 리스크 해석 표준

| 자산군 | 리스크의 근원 (Root Cause) | 노출액 (Exposure) 정의 |
| :--- | :--- | :--- |
| **PF** | 사업 성공 및 분양 현금흐름 실패 | 대출 잔액 + 미인출 한도 |
| **NPL** | 담보 처분 및 회수 현금흐름 지연 | 미상환 원금 잔액 (OPB) |
| **ABS** | 기초자산 Pool의 부실 및 워터폴 붕괴 | 투자 트랜치 액면가 |
| **Equity** | 시장 가격 변동 및 Exit 실현 실패 | 현재 시장 가치 (Market Value) |

### ─────────────

## ⚙️ 데이터 소유권 및 거버넌스

### 데이터 무결성 원칙
- **일관성**: 동일 딜 내의 포지션들은 동일한 비즈니스 시나리오를 공유해야 합니다.
- **추적성**: 모든 리스크 결과값은 어떤 현금흐름 데이터에서 파생되었는지 추적 가능해야 합니다.
