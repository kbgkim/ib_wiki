# IB Wiki 스타일 가이드 (Style Guide)

이 문서는 IB 리스크 위키의 일관성을 유지하기 위한 문서 작성 표준을 정의합니다.

## 1. 제목 및 문서 구조
- **제목**: `# 주제목 (English Title)` 형식을 사용합니다.
- **섹션 아이콘 (Emoji)**: 각 섹션에는 정의된 Emoji를 사용하여 명확한 시각적 구분을 제공합니다.
    - 🔥 **목적**: 문서의 존재 이유 및 정의
    - 📌 **개념**: 핵심 철학 및 비즈니스 정의
    - 🧠 **구조 역할**: 상위/하위 모델 간의 논리적 흐름
    - 📊 **테이블 구조**: 데이터 스키마 상세 명세
    - ⚙️ **핵식 설계 원칙**: 기술적 혹은 수식적 규칙
    - ⚖️ **리스크 산출 방식**: 계산 로직 (Credit vs Market)
    - 💰 **Cashflow 관점**: 현금흐름 기반 해석

## 2. 시각적 표현 규칙
- **구분선**: 주요 섹션 사이에는 "### ─────────────" 또는 빈 줄을 사용하여 물리적인 구분을 줍니다. (마크다운 렌더러 호환성을 위해 `---` 사용 안함)
- **강조**: 핵심 핵심 요약 문장 앞에는 `👉` (U+1F449) 이모지를 사용하여 독자의 주의를 끕니다.
- **정렬**: 표(Table)나 수식(Code Block)은 가급적 간결하게 작성하여 가독성을 높입니다.

## 3. 연결 및 참조 (Links)
- **위치**: 모든 문서는 최하단에 `## 🔗 연결` 섹션을 두어야 합니다.
- **링크 형식**: 표준 마크다운 링크 `[이름]{상대경로}`를 사용합니다.
- **순서**: 상위 모델 → 하위 데이터/스키마 순으로 배치하는 것을 권장합니다.

## 4. 용어 표준
- **Position**: 리스크 관리의 최소 기준 단위 (대문자 P 권장)
- **Cashflow**: 현금흐름 (C 대문자 권장)
- **Shortfall**: 예상 대비 부족액
- **EL**: 기대손실 (Expected Loss)
