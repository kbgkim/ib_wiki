# 통합 리스크 모델 설계 (NPL / ABS / PF / Equity)

## 1. 문제 정의

각 자산 유형은 서로 다른 리스크 구조를 가짐

  자산     리스크 구조
  -------- --------------------
  Equity   가격 변동
  ABS      현금흐름 안정성
  PF       프로젝트 성공 여부
  NPL      회수 가능성

→ 그대로는 하나의 모델로 통합 불가능

------------------------------------------------------------------------

## 2. 핵심 아이디어

모든 자산을 다음으로 환원:

> 미래 현금흐름의 불확실성

    모든 자산 → Cashflow → 확률 → 손실

------------------------------------------------------------------------

## 3. 공통 리스크 프레임워크

-   PD (Probability of Default)
-   LGD (Loss Given Default)
-   EAD (Exposure at Default)

### 공식

    Expected Loss = PD × LGD × EAD

------------------------------------------------------------------------

## 4. 자산별 매핑

### NPL

-   PD = 100%
-   LGD = 회수율 기반
-   EAD = 채권 잔액

### ABS

-   PD = underlying asset default
-   LGD = tranche 구조 영향
-   EAD = 투자 금액

### PF

-   PD = 프로젝트 실패 확률
-   LGD = 담보 가치
-   EAD = 대출 규모

### Equity

-   PD = 기업 실패 확률
-   LGD ≈ 100%
-   EAD = 투자 금액

------------------------------------------------------------------------

## 5. 데이터 모델

### Asset

    asset_id
    asset_type
    exposure
    cashflow_profile

### RiskFactor

    asset_id
    PD
    LGD
    scenario

### Cashflow

    asset_id
    expected_cf
    stress_cf
    timing

------------------------------------------------------------------------

## 6. 리스크 엔진 흐름

    Kafka → Spark → Risk Calculation → DW

    Asset → Cashflow → Scenario → PD/LGD → Loss

------------------------------------------------------------------------

## 7. 시나리오 기반 분석

  Scenario   설명
  ---------- ------
  Base       정상
  Stress     위기
  Worst      극단

------------------------------------------------------------------------

## 8. 상관관계

-   경기 하락 → NPL 증가
-   금리 상승 → PF 악화
-   시장 하락 → Equity 손실

→ Correlation Matrix 적용

------------------------------------------------------------------------

## 9. 핵심 설계 철학

1.  자산이 아니라 리스크를 통합
2.  Cashflow 중심 모델
3.  Scenario 기반 분석

------------------------------------------------------------------------

## 10. 결론

모든 자산은 서로 다르지만

> Cashflow + PD/LGD + Scenario

로 통합 가능

→ 포트폴리오 단위 리스크 계산 가능
